# Настройка WIS2 Downloader на виртуальной машине студента

!!! abstract "Результаты обучения!"

    По завершении практического занятия вы сможете:

    - настроить собственный экземпляр "WIS2 Downloader" и управлять необходимыми конфигурациями
    - работать с экземпляром и настраивать подписки
    - удалять подписки и находить загруженные данные

## Введение

На этом занятии вы узнаете, как настроить экземпляр WIS2 Downloader на предоставленной виртуальной машине студента и как работать с его различными сервисами.

!!! note "О WIS2 Downloader"
     
     WIS2 Downloader доступен как автономный проект Docker Compose и рекомендуется запускать его на отдельном сервере от wis2box, чтобы загрузки не мешали публикации сообщений.

     Если вы хотите разработать собственный сервис для подписки на уведомления WIS2 и загрузки данных, вы можете использовать [исходный код WIS2 Downloader](https://github.com/World-Meteorological-Organization/wis2downloader) в качестве примера.

## Подготовка и требования

!!! note "Если не во время тренинга"

    Следующие шаги следует выполнять только в том случае, если упомянутые порты недоступны на сервере по умолчанию. В любой конфигурации это единственные порты, которые необходимо открыть для использования всех возможностей стека WIS2 Downloader.    

Перед началом работы войдите в свою виртуальную машину студента, убедившись, что вы настроили туннелирование через SSH для следующих портов:

- `5002 (API)`
- `8080 (UI)`
- `3000 (Grafana)`

Для этого вы можете изменить настройки подключения в Putty, добавив сопоставление 3 портов с портами на вашем компьютере (localhost):

![adding tunnels in putty](../assets/img/putty-add-tunnel.png)

!!! note "Туннелирование на Linux и macOS"

    На Linux и macOS вы можете настроить те же туннели прямо из терминала, используя флаг `-L` в вашей SSH-команде:

    ```bash
    ssh -L 5002:localhost:5002 -L 8080:localhost:8080 -L 3000:localhost:3000 <username>@<WIS2DOWNLOADER_BASE_URL>
    ```

    Замените `<username>` и `<WIS2DOWNLOADER_BASE_URL>` на учетные данные вашей виртуальной машины студента.

## Установка WIS2 Downloader

Скачайте последний релиз tarball с GitHub и извлеките его на своей виртуальной машине студента:

```bash
wget https://github.com/World-Meteorological-Organization/wis2downloader/archive/refs/tags/v1.0.0b1+rc4.tar.gz
tar -xzf v1.0.0b1+rc4.tar.gz
cd wis2downloader-*
```

Запустите скрипт настройки для создания файла конфигурации:

```bash
bash setup.sh
```

Используйте следующий путь для загрузки `/home/<username>/wis2-downloads`, заменив `<username>` на ваше имя пользователя. Затем нажмите Enter, чтобы использовать значения по умолчанию для пользователя и групп.

!!! note "Управление разрешениями пользователя"
    Вы можете использовать другие значения пользователя и группы, изменив `WIS2DWONLOADER_UID` и `WIS2DWONLOADER_GID` в файле .env.
    Не забудьте пересобрать образы после внесения изменений, чтобы они вступили в силу. 

Этот процесс создаст файл `.env` с настройками по умолчанию и случайными значениями для `FLASK_SECRET_KEY` и `REDIS_PASSWORD`. Вы можете просмотреть файл с помощью команды `cat .env` — настройки по умолчанию подходят для развертывания на одной машине.

Запустите полный стек сервисов:

```bash
docker compose up -d
```

!!! note "Проверка запущенных контейнеров"
    Вы можете убедиться, что все контейнеры успешно запущены, с помощью команды:
    ```bash
    docker compose ps
    ```
    Вы должны увидеть сервисы для менеджера подписок, MQTT подписчиков, UI, Celery workers, Redis, Prometheus, Grafana и Loki.

## Доступ к интерфейсу WIS2 Downloader

Откройте веб-браузер и перейдите к интерфейсу вашего экземпляра WIS2 Downloader, набрав `http://<WIS2DOWNLOADER_BASE_URL>:8080`.

Вы окажетесь на главной странице, которая по умолчанию отображает вид `Dashboard` с панелью Grafana.

![WIS2 Downloader Landing Page](../assets/img/wis2-downloader-landing-page.png)

В левом боковом меню вы сможете перейти ко всем различным разделам интерфейса.

Основные доступные разделы:

- **Dashboard** — главная страница по умолчанию, встроенная панель Grafana, отображающая активность загрузки, статус очереди и метрики работающего сервиса. Также доступно по адресу `http://<WIS2DOWNLOADER_BASE_URL>:3000`.
- **Catalogue View** — просмотр доступных наборов данных WIS2 с помощью поиска или фильтрации глобального каталога. Выберите тему и директорию для сохранения, затем нажмите *Subscribe*, чтобы начать загрузку.
- **Tree View** — навигация по иерархии тем WIS2 в виде раскрывающегося дерева. Полезно для изучения доступных тем перед подпиской.
- **Manual Subscribe** — создание подписки путем ввода деталей темы напрямую, без использования глобальных каталогов. Полезно для более свободной подписки на темы с использованием любых необходимых подстановочных знаков и доступа к темам, которые не отображаются в GDC, например, шлюзы GTS и темы, опубликованные на частных брокерах в нестандартных конфигурациях.
- **Manage Subscriptions** — просмотр и управление всеми активными подписками. Здесь вы можете увидеть, какие темы отслеживаются, и удалить те, которые больше не нужны.
- **Settings** — в настоящее время позволяет перезагрузить каталог наборов данных из глобальных каталогов. Этот раздел будет расширен в будущих версиях для охвата общей конфигурации и управления WIS2 Downloader.
- **Documentation** — встроенная документация для WIS2 Downloader.

## Управление подписками через интерфейс

Как и в предыдущем примере, вы получите доступ к интерфейсу работающего экземпляра, перейдя по адресу `http://<WIS2DOWNLOADER_BASE_URL>:8080`.

Существует 3 способа настройки подписки:

- В **Catalogue View**, просматривая доступные темы аналогично порталам GDC.
- В **Tree View**, выбирая тему из каталога GDC, исследуя темы, как в MQTT Explorer.
- В **Manual Subscribe**, где вы можете вручную вводить желаемые темы, фильтры и другие параметры.

Для следующего упражнения мы подпишемся на все уведомления synop, поступающие от всех узлов WIS2:

- Сначала перейдите в **Manual Subscribe**.
- Введите тему как `cache/a/wis2/+/data/core/weather/surface-based-observations/synop`
- Установите папку назначения как `synop-data`

Результат должен выглядеть следующим образом:
![WIS2 Downloader Manual Subscribe](../assets/img/wis2-downloader-manual-subscribe.png)

Теперь нажмите кнопку **Subscribe** и подтвердите свою подписку.

После этого проверьте папку загрузки на вашей виртуальной машине студента, используя команду:

```bash
ls -R ~/wis2-downloads
```

Теперь вы должны увидеть серию файлов, загруженных вашим экземпляром.

На последнем этапе мы можем удалить подписку, перейдя в раздел **Manage Subscriptions** и нажав кнопку **Unsubscribe**.

![WIS2 Downloader Delete Subscription](../assets/img/wis2-downloader-delete-subscription.png)

!!! note "Удаление загруженных файлов"

    Рекомендуется очистить папку загрузок после завершения упражнения, чтобы освободить место на виртуальной машине студента. Для этого выполните следующую команду для удаления файлов предыдущего упражнения.

    ```bash
    rm -fr ~/wis2-downloads/synop-data
    ```

## Просмотр конфигурации WIS2 Downloader

Экземпляр WIS2 Downloader можно настроить с помощью переменных окружения, определенных в вашем файле `.env`.

Вы можете ознакомиться с описанием переменных окружения в [Руководстве администратора WIS2 Downloader, раздел 2.1](https://world-meteorological-organization.github.io/wis2downloader/en/admin-guide.html)

Чтобы просмотреть текущую конфигурацию WIS2 Downloader, используйте следующую команду:

```bash
cat .env
```

!!! question "Просмотр конфигурации WIS2 Downloader"

    Каков период хранения загруженных данных по умолчанию?

    На каком порту работает API менеджера подписок?

??? success "Нажмите, чтобы увидеть ответ"

    Период хранения загруженных данных по умолчанию составляет `30` дней, как указано в `DOWNLOAD_RETENTION_PERIOD`.

    API менеджера подписок работает на порту `5002`, как указано в `WIS2DOWNLOADER_SUBSCRIPTION_MANAGER_URL`.

!!! note "Обновление конфигурации WIS2 Downloader"

    Чтобы обновить конфигурацию, отредактируйте файл `.env` и перезапустите стек, чтобы изменения вступили в силу:

    ```bash
    docker compose up -d
    ```

Вы можете оставить конфигурацию по умолчанию для следующих упражнений.

## API WIS2 Downloader

WIS2 Downloader предоставляет REST API по адресу `<WIS2DOWNLOADER_BASE_URL>:5002`. Убедитесь, что сервис готов:

```bash
curl localhost:5002/health
```

Вы должны увидеть:

```json
{"status": "healthy"}
```

Чтобы создать подписку, отправьте запрос `POST` с указанием MQTT `topic` и необязательной поддиректории `target`, где будут сохраняться файлы:

```bash
curl -s -X POST localhost:5002/subscriptions \
  -H "Content-Type: application/json" \
  -d '{"topic": "cache/a/wis2/+/data/core/weather/surface-based-observations/#", "target": "surface-obs"}'
```

Как и раньше, загруженные файлы можно просмотреть, проверив папку `surface-obs` в директории загрузки:

```bash
ls -R ~/wis2-downloads/surface-obs
```

Ответ включает UUID, назначенный новой подписке. Используйте его для удаления подписки, когда она больше не нужна:

```bash
curl -X DELETE localhost:5002/subscriptions/{id}
```

!!! note "Удаление загруженных файлов"

    Рекомендуется очистить папку загрузок после завершения упражнения, чтобы освободить место на виртуальной машине студента. Для этого выполните следующую команду для удаления файлов предыдущего упражнения.

    ```bash
    rm -fr ~/wis2-downloads/surface-obs
    ```

Полный список доступных конечных точек (список, получение, обновление подписок и другие) можно найти в интерактивной документации Swagger, доступной по адресу `localhost:5002/swagger`.
