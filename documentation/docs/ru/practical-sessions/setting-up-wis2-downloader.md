# Настройка WIS2 Downloader на виртуальной машине студента

!!! abstract "Результаты обучения!"

    По завершении данного практического занятия вы сможете:

    - настроить собственный экземпляр "WIS2 Downloader" и управлять необходимыми конфигурациями
    - ориентироваться в экземпляре и использовать его различные возможности

## Введение

На этом занятии вы узнаете, как настроить экземпляр WIS2 Downloader на предоставленной виртуальной машине студента и как работать с его различными сервисами.

!!! note "О WIS2 Downloader"
     
     WIS2 Downloader доступен как автономный проект Docker Compose и рекомендуется запускать его на отдельном сервере от wis2box, чтобы загрузки не мешали публикации сообщений.

     Если вы хотите разработать собственный сервис для подписки на уведомления WIS2 и загрузки данных, вы можете использовать [исходный код WIS2 Downloader](https://github.com/World-Meteorological-Organization/wis2downloader) в качестве примера.

## Подготовка и требования

!!! note "Если не во время тренинга"

    Следующие шаги следует выполнять только в том случае, если указанные порты по умолчанию недоступны на сервере. В любой конфигурации это единственные порты, которые необходимо открыть для использования всех возможностей стека WIS2 Downloader.    

Перед началом работы войдите на вашу виртуальную машину студента, убедившись, что вы настроили туннель через SSH для следующих портов:

- `5002 (API)`
- `8080 (UI)`
- `3000 (Grafana)`

Для этого вы можете изменить настройки подключения в Putty:

![access putty tunnel settings](../assets/img/putty-tunnel-settings.png)

Затем добавьте сопоставление 3 портов с портами на вашем компьютере (localhost):

![adding tunnels in putty](../assets/img/putty-add-tunnel.png)

## Установка WIS2 Downloader

Скачайте последний архив релиза с GitHub и извлеките его на вашей виртуальной машине студента:

```bash
wget https://github.com/World-Meteorological-Organization/wis2downloader/releases/latest/download/wis2downloader-latest.tar.gz
tar -xzf wis2downloader-latest.tar.gz
cd wis2downloader-*
```

Запустите скрипт настройки для создания файла конфигурации:

```bash
bash setup.sh
```

Это создаст файл `.env` с настройками по умолчанию и сгенерирует случайные значения для `FLASK_SECRET_KEY` и `REDIS_PASSWORD`. Вы можете просмотреть файл с помощью команды `cat .env` — настройки по умолчанию подходят для развертывания на одной машине.

Установите плагин Docker Loki, используемый для отправки логов:

```bash
ARCH=$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/')
docker plugin install grafana/loki-docker-driver:3.6.7-${ARCH} --alias loki --grant-all-permissions
```

Убедитесь, что плагин включен:

```bash
docker plugin ls
```

Вы должны увидеть `loki:latest` с `ENABLED`, установленным в `true`.

Создайте выделенную группу `wis2`, добавьте в нее вашего пользователя и настройте файл `.env` и каталог загрузок соответствующим образом:

```bash
sudo groupadd wis2
sudo usermod -aG wis2 $USER
sed -i "s/^UID=.*/UID=$(id -u)/" .env
sed -i "s/^GID=.*/GID=$(getent group wis2 | cut -d: -f3)/" .env
mkdir -p downloads
sudo chown $(id -un):wis2 downloads
chmod 775 downloads
```

!!! note "Требуется повторный вход"
    Изменение членства в группе вступает в силу только после выхода и повторного входа в вашу SSH-сессию.

Запустите полный стек сервисов:

```bash
docker compose up -d
```

Подождите около 30 секунд, пока проверки состояния не завершатся, затем убедитесь, что менеджер подписок готов:

```bash
curl http://localhost:5002/health
```

!!! note "Проверка запущенных контейнеров"
    Вы можете убедиться, что все контейнеры успешно запущены, с помощью команды:
    ```bash
    docker compose ps
    ```
    Вы должны увидеть сервисы для менеджера подписок, подписчиков MQTT, UI, рабочих Celery, Redis, Prometheus, Grafana и Loki.

### Доступ к UI WIS2 Downloader

Откройте веб-браузер и перейдите к UI вашего экземпляра WIS2 Downloader, введя `http://localhost:8080`.

Вы окажетесь на стартовой странице, которая по умолчанию настроена на раздел `Help`, где отображается документация.

![WIS2 Downloader Landing Page](../assets/img/wis2-downloader-landing-page.png)

В меню боковой панели слева вы сможете перейти ко всем различным разделам UI.

Основные доступные разделы:

- **Dashboard** — встроенная панель Grafana, показывающая активность загрузки, статус очереди и метрики работающего сервиса. Также доступна по адресу `http://localhost:3000`.
- **Catalogue View** — просмотр доступных наборов данных WIS2 с помощью поиска или фильтрации глобального каталога. Выберите тему и каталог для сохранения, затем нажмите *Subscribe*, чтобы начать загрузку.
- **Tree View** — навигация по иерархии тем WIS2 в виде сворачиваемого дерева. Полезно для изучения доступных тем перед подпиской.
- **Manual Subscription** — создание подписки путем ввода темы и данных брокера напрямую, без использования глобальных каталогов обнаружения. Полезно для подписки на темы от конкретных узлов WIS2 или частных брокеров.
- **Subscriptions** — просмотр и управление всеми активными подписками. Здесь вы можете увидеть, какие темы отслеживаются, и удалить те, которые больше не нужны.
- **Settings** — в настоящее время позволяет перезагрузить каталог наборов данных из глобальных каталогов обнаружения. Этот раздел будет расширен в будущих выпусках для охвата общей конфигурации и управления WIS2 Downloader.
- **Help** — стартовая страница по умолчанию, показывающая встроенную документацию для WIS2 Downloader.

### Проверка конфигурации WIS2 Downloader

Экземпляр WIS2 Downloader можно настроить с помощью переменных окружения, определенных в вашем файле `.env`.

Вы можете ознакомиться с описанием переменных окружения в [разделе 2.1 Руководства администратора WIS2 Downloader](https://world-meteorological-organization.github.io/wis2downloader/en/admin-guide.html).

Чтобы проверить текущую конфигурацию WIS2 Downloader, используйте следующую команду:

```bash
cat .env
```

!!! question "Проверка конфигурации WIS2 Downloader"

    Каков период хранения загруженных данных по умолчанию?

    На каком порту API менеджера подписок слушает запросы?

??? success "Нажмите, чтобы увидеть ответ"

    Период хранения загруженных данных по умолчанию составляет `30` дней, как указано в `DOWNLOAD_RETENTION_PERIOD`.

    API менеджера подписок слушает запросы на порту `5002`, как указано в `WIS2DOWNLOADER_SUBSCRIPTION_MANAGER_URL`.

!!! note "Обновление конфигурации WIS2 Downloader"

    Чтобы обновить конфигурацию, отредактируйте файл `.env` и перезапустите стек для применения изменений:

    ```bash
    docker compose up -d
    ```

Вы можете оставить конфигурацию по умолчанию для следующих упражнений.

### API WIS2 Downloader

WIS2 Downloader предоставляет REST API по адресу `<WIS2DOWNLOADER_BASE_URL>:5002/api`. Убедитесь, что сервис готов:

```bash
curl <WIS2DOWNLOADER_BASE_URL>:5002/api/health
```

Вы должны увидеть:

```json
{"status": "healthy"}
```

Чтобы создать подписку, отправьте запрос `POST` с указанием MQTT `topic` и необязательного подкаталога `target`, куда будут сохраняться файлы:

```bash
curl -s -X POST <WIS2DOWNLOADER_BASE_URL>:5002/api/subscriptions \
  -H "Content-Type: application/json" \
  -d '{"topic": "cache/a/wis2/+/data/core/weather/surface-based-observations/#", "target": "surface-obs"}'
```

Ответ включает UUID, назначенный новой подписке. Используйте его для удаления подписки, когда она больше не нужна:

```bash
curl -X DELETE <WIS2DOWNLOADER_BASE_URL>:5002/api/subscriptions/{id}
```

Для полного списка доступных конечных точек (список, получение, обновление подписок и другие), обратитесь к интерактивной документации Swagger, доступной по адресу `<WIS2DOWNLOADER_BASE_URL>:5002/api/openapi`.
