# Настройка WIS2 Downloader на виртуальной машине студента

!!! abstract "Результаты обучения!"

    По окончании практического занятия вы сможете:

    - настроить собственный экземпляр "WIS2 Downloader" и управлять необходимыми конфигурациями
    - ориентироваться в экземпляре и использовать его различные возможности

## Введение

На этом занятии вы узнаете, как настроить экземпляр WIS2 Downloader на предоставленной виртуальной машине студента и как работать с его различными сервисами.

!!! note "О WIS2 Downloader"
     
     WIS2 Downloader доступен как отдельный проект Docker Compose и рекомендуется запускать его на отдельном сервере от wis2box, чтобы загрузки не мешали публикации сообщений.

     Если вы хотите разработать собственный сервис для подписки на уведомления WIS2 и загрузки данных, вы можете использовать [исходный код WIS2 Downloader](https://github.com/World-Meteorological-Organization/wis2downloader) в качестве примера.

## Подготовка и требования

!!! note "Если не во время тренинга"

    Следующие шаги следует выполнять только в том случае, если указанные порты по умолчанию недоступны на сервере. В любой конфигурации это единственные порты, которые необходимо открыть для использования всех возможностей стека WIS2 Downloader.    

Перед началом работы войдите в свою виртуальную машину студента, убедившись, что вы настроили туннель через SSH для следующих портов:

- `5002 (API)`
- `8080 (UI)`
- `3000 (Grafana)`

Для этого вы можете изменить настройки подключения в Putty:

![access putty tunnel settings](../assets/img/putty-tunnel-settings.png)

Затем добавьте сопоставление 3 портов с портами на вашем собственном ПК (localhost):

![adding tunnels in putty](../assets/img/putty-add-tunnel.png)

## Установка WIS2 Downloader

Скачайте последний tarball-релиз с GitHub и распакуйте его на своей виртуальной машине студента:

```bash
wget https://github.com/World-Meteorological-Organization/wis2downloader/releases/latest/download/wis2downloader-latest.tar.gz
tar -xzf wis2downloader-latest.tar.gz
cd wis2downloader-*
```

Запустите скрипт настройки для создания файла конфигурации:

```bash
bash setup.sh
```

Это создаст файл `.env` с настройками по умолчанию и случайными значениями для `FLASK_SECRET_KEY` и `REDIS_PASSWORD`. Вы можете просмотреть файл с помощью команды `cat .env` — настройки по умолчанию подходят для развертывания на одной машине.

Установите плагин Docker Loki, используемый для отправки логов:

```bash
ARCH=$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/')
docker plugin install grafana/loki-docker-driver:3.6.7-${ARCH} --alias loki --grant-all-permissions
```

Проверьте, что плагин включен:

```bash
docker plugin ls
```

Вы должны увидеть `loki:latest` в списке с `ENABLED`, установленным в `true`.

Создайте выделенную группу `wis2`, добавьте своего пользователя в нее и настройте файл `.env` и каталог загрузок соответствующим образом:

```bash
sudo groupadd wis2
sudo usermod -aG wis2 $USER
sed -i "s/^UID=.*/UID=$(id -u)/" .env
sed -i "s/^GID=.*/GID=$(getent group wis2 | cut -d: -f3)/" .env
mkdir -p downloads
sudo chown $(id -un):wis2 downloads
chmod 775 downloads
```

!!! note "Требуется повторный вход в систему"
    Изменения членства в группе вступают в силу только после выхода и повторного входа в вашу SSH-сессию.

Запустите полный стек сервисов:

```bash
docker compose up -d
```

Подождите около 30 секунд, пока проверки состояния не завершатся, затем убедитесь, что менеджер подписок готов:

```bash
curl http://<WIS2DOWNLOADER_BASE_URL>:5002/health
```

!!! note "Проверка запущенных контейнеров"
    Вы можете проверить, что все контейнеры успешно запущены, с помощью команды:
    ```bash
    docker compose ps
    ```
    Вы должны увидеть сервисы для менеджера подписок, подписчиков MQTT, UI, рабочих Celery, Redis, Prometheus, Grafana и Loki.

### Доступ к интерфейсу WIS2 Downloader

Откройте веб-браузер и перейдите к интерфейсу вашего экземпляра WIS2 Downloader, набрав `http://<WIS2DOWNLOADER_BASE_URL>:8080`.

Вы окажетесь на главной странице, которая по умолчанию открывает раздел `Help`, где отображается документация.

![WIS2 Downloader Landing Page](../assets/img/wis2-downloader-landing-page.png)

В левом боковом меню вы сможете перейти ко всем различным разделам интерфейса.

Основные доступные разделы:

- **Dashboard** — встроенная панель Grafana, отображающая активность загрузок, статус очереди и метрики работающего сервиса. Также доступна по адресу `http://<WIS2DOWNLOADER_BASE_URL>:3000`.
- **Catalogue View** — просмотр доступных наборов данных WIS2 с помощью поиска или фильтрации глобального каталога. Выберите тему и каталог для сохранения, затем нажмите *Subscribe*, чтобы начать загрузку.
- **Tree View** — навигация по иерархии тем WIS2 в виде сворачиваемого дерева. Полезно для изучения доступных тем перед подпиской.
- **Manual Subscribe** — создание подписки путем ввода темы и данных брокера напрямую, без использования глобальных каталогов. Полезно для подписки на темы от конкретных узлов WIS2 или частных брокеров.
- **Manage Subscriptions** — просмотр и управление всеми активными подписками. Здесь вы можете увидеть, какие темы отслеживаются, и удалить те, которые больше не нужны.
- **Settings** — в настоящее время позволяет перезагрузить каталог наборов данных из глобальных каталогов. В будущих версиях этот раздел будет расширен для общей конфигурации и управления WIS2 Downloader.
- **Help** — раздел по умолчанию, отображающий встроенную документацию для WIS2 Downloader.

### Управление подписками через интерфейс

Как и в предыдущем примере, вы получите доступ к интерфейсу работающего экземпляра, перейдя на `http://<WIS2DOWNLOADER_BASE_URL>:8080`.

Оттуда есть три способа настроить подписку:

- В **Catalogue View**, просматривая доступные темы аналогично порталам GDC.
- В **Tree View**, выбирая тему из каталога GDC, исследуя темы, как в MQTT Explorer.
- В **Manual Subscribe**, где вы можете вручную ввести желаемые темы, фильтры и другие параметры.

Для следующего упражнения мы подпишемся на уведомления, поступающие от шлюза GTS к WIS2, управляемого DWD:

- Сначала перейдите в **Manual Subscribe**.
- Введите тему как `cache/a/wis2/de-dwd-gts-to-wis2/data/core/#`
- Установите папку назначения как `gts-data`

Окончательный результат должен выглядеть примерно так:
![WIS2 Downloader Manual Subscribe](../assets/img/wis2-downloader-manual-subscribe.png)

После этого перейдите в папку загрузок на вашей виртуальной машине студента, используя команды:

```bash
ls -R wisdownloader/downloads
```

Теперь вы должны увидеть серию файлов, которые были загружены вашим экземпляром.

На последнем этапе мы можем удалить подписку, перейдя в раздел **Manage Subscriptions** и нажав кнопку **Unsubscribe**.

![WIS2 Downloader Delete Subscription](../assets/img/wis2-downloader-delete-subscription.png)

!!! note "Удаление загруженных файлов"

    Рекомендуется очистить папку загрузок после завершения упражнения, чтобы освободить место на виртуальной машине студента. Для этого выполните следующую команду, чтобы удалить файлы предыдущих упражнений.

    ```bash
    rm -fr wisdownloader/downloads
    ```

### Обзор конфигурации WIS2 Downloader

Экземпляр WIS2 Downloader можно настроить с помощью переменных окружения, определенных в вашем файле `.env`.

Вы можете ознакомиться с описанием переменных окружения в [Разделе 2.1 Руководства администратора WIS2 Downloader](https://world-meteorological-organization.github.io/wis2downloader/en/admin-guide.html)

Чтобы просмотреть текущую конфигурацию WIS2 Downloader, вы можете использовать следующую команду:

```bash
cat .env
```

!!! question "Просмотр конфигурации WIS2 Downloader"

    Каков период хранения загруженных данных по умолчанию?

    На каком порту слушает API менеджера подписок?

??? success "Нажмите, чтобы увидеть ответ"

    Период хранения загруженных данных по умолчанию составляет `30` дней, как указано в `DOWNLOAD_RETENTION_PERIOD`.

    API менеджера подписок слушает порт `5002`, как указано в `WIS2DOWNLOADER_SUBSCRIPTION_MANAGER_URL`.

!!! note "Обновление конфигурации WIS2 Downloader"

    Чтобы обновить конфигурацию, отредактируйте файл `.env` и перезапустите стек, чтобы применить изменения:

    ```bash
    docker compose up -d
    ```

Вы можете оставить конфигурацию по умолчанию для следующих упражнений.

### API WIS2 Downloader

WIS2 Downloader предоставляет REST API по адресу `<WIS2DOWNLOADER_BASE_URL>:5002/api`. Убедитесь, что сервис готов:

```bash
curl <WIS2DOWNLOADER_BASE_URL>:5002/api/health
```

Вы должны увидеть:

```json
{"status": "healthy"}
```

Чтобы создать подписку, отправьте запрос `POST` с темой MQTT `topic` и необязательной поддиректорией `target`, где будут сохраняться файлы:

```bash
curl -s -X POST <WIS2DOWNLOADER_BASE_URL>:5002/api/subscriptions \
  -H "Content-Type: application/json" \
  -d '{"topic": "cache/a/wis2/+/data/core/weather/surface-based-observations/#", "target": "surface-obs"}'
```

Ответ включает UUID, назначенный новой подписке. Используйте его, чтобы удалить подписку, когда она больше не нужна:

```bash
curl -X DELETE <WIS2DOWNLOADER_BASE_URL>:5002/api/subscriptions/{id}
```

Для полного списка доступных конечных точек (список, получение, обновление подписок и другое) обратитесь к интерактивной документации Swagger, доступной по адресу `<WIS2DOWNLOADER_BASE_URL>:5002/api/openapi`.
